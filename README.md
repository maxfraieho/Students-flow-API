# StudentFlow — Monorepo

```
/               ← React + Vite frontend (managed by Lovable.dev)
/backend        ← FastAPI Python backend  (runs on Raspberry Pi)
/worker         ← Cloudflare Worker API gateway
```

## Architecture

```
Browser  (React frontend at repo root)
   │  Authorization: Bearer <OPERATOR_TOKEN>
   ▼
Cloudflare Worker  (/worker)
   │  Validates Bearer token → strips Authorization header
   │  Proxies over HTTPS via cloudflared tunnel
   ▼
FastAPI Backend  (/backend)          port 8000
   │  SQLite  ·  OS keyring  ·  GitPython
   ▼  (runs on Raspberry Pi)
Student GitHub repos  (git push --force per broadcast)
```

## `/backend` — FastAPI Python Backend

| File / Dir | Purpose |
|------------|---------|
| `app/` | Application code (models, services, routers, git client) |
| `app/routers/sync.py` | SSE broadcast endpoint `GET /api/sync/broadcast` |
| `app/services/sync_service.py` | `broadcast_canonical_stream()` generator |
| `app/db/seeds.py` | Seed 20 students — reads PAT from `GITHUB_TOKEN` env var |
| `requirements.txt` | Python dependencies |
| `run.py` | Uvicorn entry point |
| `SETUP_PI.md` | Full Raspberry Pi deployment guide |
| `.env.example` | Environment variable template (copy → `.env`, git-ignored) |

### Run locally (Replit)

Use the **"Backend (FastAPI)"** workflow, or from a shell:

```bash
cd backend
pip install -r requirements.txt
python run.py                   # starts uvicorn on port 8000
```

### Seed the database

```bash
cd backend
export GITHUB_TOKEN="YOUR_GITHUB_TOKEN_HERE"   # set in Replit Secrets or shell
python -m app.db.seeds
```

### Deploy to Raspberry Pi

See [`backend/SETUP_PI.md`](backend/SETUP_PI.md).

## `/worker` — Cloudflare Worker API Gateway

| File | Purpose |
|------|---------|
| `worker.js` | Bearer-token auth + tunnel reverse proxy |
| `wrangler.toml` | Wrangler config (`TUNNEL_URL`, `FRONTEND_URL`) |
| `DEPLOY.md` | Step-by-step deployment guide |

### Deploy

```bash
cd worker
# 1. Edit wrangler.toml → set TUNNEL_URL and FRONTEND_URL
# 2. Store the secret (never in wrangler.toml)
wrangler secret put OPERATOR_TOKEN
# 3. Deploy
wrangler deploy
```

## `/` — React + Vite Frontend (Lovable.dev)

The repo root is reserved for a React/Vite frontend generated by Lovable.dev.
Lovable will create `src/`, `index.html`, `vite.config.ts`, `package.json`, etc. here.

All backend/worker code is isolated in `/backend` and `/worker` so it does not
interfere with frontend tooling.

## Environment Variables

| Variable | Where set | Purpose |
|----------|-----------|---------|
| `GITHUB_TOKEN` | Replit Secrets / `.env` | GitHub PAT for seeding + broadcast push |
| `OPERATOR_TOKEN` | Cloudflare Worker Secret | Auth token checked by the Worker |
| `TUNNEL_URL` | `worker/wrangler.toml` `[vars]` | cloudflared tunnel URL → Pi |
| `FRONTEND_URL` | `worker/wrangler.toml` `[vars]` | Lovable app origin (CORS allow-list) |

> **Never commit real values** — use Replit Secrets, `wrangler secret put`, or a
> local `.env` file (which is git-ignored).

## SSE Broadcast Endpoint

`GET /api/sync/broadcast` streams progress as Server-Sent Events:

```
data: {"event": "canonical_pulled", "branch": "main", "status": "ok"}

data: {"event": "broadcast_start", "total": 20}

data: {"event": "processing", "student": "Emily Johnson", "index": 1, "total": 20}

data: {"event": "done", "student": "Emily Johnson", "index": 1, "status": "success", "commit": "a1b2c3d4"}

data: {"event": "broadcast_complete", "total": 20, "succeeded": 19, "failed": 1}
```

The Cloudflare Worker proxies this stream without buffering (`X-Accel-Buffering: no`).
